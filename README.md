## What Is This?

Most developers use AI coding agents as glorified autocomplete. This guide shows you how to set them up as a **role-separated development team** — with planning, implementation, review, QA, and deployment each handled by specialized workflows.

The guide covers the full stack from scratch:

1. **Foundation** — Terminal, shell, Homebrew, Node.js, editor setup
2. **Claude Code** — Installation, configuration, CLAUDE.md best practices, sub-agent workflows
3. **Codex** — OpenAI's CLI agent as a second opinion and code reviewer
4. **MCP Servers** — Connecting agents to browsers, GitHub, Slack, and more
5. **gstack** — Skill-based workflow engine that enforces dev process structure
6. **CLAUDE.md / AGENTS.md** — Production-grade prompt configuration
7. **Ralph (Ouroboros)** — Autonomous spec-first development loop
8. **Appendix** — Claude Code internals, `.claude/` directory structure, hooks, permissions

## Who Is This For?

- **Founders / Tech Leads** who want structured AI-assisted development, not chaos
- **Developers** exploring Claude Code, Codex, or gstack for the first time
- **Anyone** who wants to go beyond "chat with AI" to "run AI like a team"
